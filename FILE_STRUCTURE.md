## 🎯 Architecture Overview

```
┌─────────────────────────────────────────────────────┐
│         Browser / Next.js Client (React)            │
├─────────────────────────────────────────────────────┤
│  pages (*.tsx) → components → lib/auth utilities    │
│  styling: TailwindCSS + dark theme                 │
│  forms: Controlled inputs + validation             │
└────────────┬────────────────────────────────────────┘
             │ API Routes
             ↓
┌─────────────────────────────────────────────────────┐
│         Next.js API Routes (Node.js)                │
├─────────────────────────────────────────────────────┤
│  /api/auth/* → authentication                       │
│  /api/supporters/* → CRUD operations                │
│  Server-side: Validation + auth check               │
└────────────┬────────────────────────────────────────┘
             │ Supabase SDK
             ↓
┌─────────────────────────────────────────────────────┐
│       Supabase (PostgreSQL + Auth)                  │
├─────────────────────────────────────────────────────┤
│  Tables: users, supporters, events, etc.            │
│  Row Level Security: Role-based access              │
│  Auth: Email/password authentication                │
│ Audit: Activity logging & tracking                  │
└─────────────────────────────────────────────────────┘
```
